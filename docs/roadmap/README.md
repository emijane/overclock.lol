# Roadmap Docs

This folder is the canonical home for shipped-behavior notes, near-term product
planning, and backlog material.

## Purpose

- capture current product behavior in one stable place
- separate roadmap planning from QA and implementation audits
- keep route truth aligned with `README.md`, `overclock/README.md`, and `AGENTS.md`

## Current Sections

- `PROJECT_CONTEXT_ROADMAP.md`
  - current product snapshot, architecture notes, and roadmap direction
- `PRODUCT_BACKLOG.md`
  - concise backlog items that are still relevant after cleanup
- `lfg/*`
  - Duos / Stacks behavior notes and future LFG planning
- `matches/*`
  - Invite to Play and `/matches` planning
- `presence/*`
  - presence and availability planning
- `profiles/*`
  - public profile planning and performance notes
- `trust/*`
  - trust and verification roadmap notes
